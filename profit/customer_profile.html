1. 분석 대상 및 케이스 분류

Case 1: 배송대행 (Single)

배송 대행(WE_SHIP) 서비스를 통해 들어온 단일 패키지 대상

  • Filter: package_type='SINGLE', reference_type='ASN'
  • Data: shipment_package_new, fee_new

Case 2: 구매대행 (Single)

구매 대행(WeBuy) 서비스를 통해 결제 및 출고된 단일 패키지 대상

  • Filter: last_status != 'CANCELED', reference_type IN ('BUY_REQUEST', 'BUYFORME')
  • Data: webuy_all_market, transaction, shipment_package_new

Case 3: 배송대행 (묶음)

배송 대행 상품들로만 구성된 합배송(Console/Repack) 패키지

  • Filter: package_type IN ('CONSOLE', 'REPACK'), All Inner='ASN'
  • Structure: Case 3(상위) + Case 3-1(하위 개별)

Case 4: 구매대행 (묶음)

구매 대행 상품들로만 구성된 합배송(Console/Repack) 패키지

  • Filter: package_type IN ('CONSOLE', 'REPACK'), All Inner='BUY_REQUEST/BUYFORME'
  • Structure: Case 4(상위 합산) + Case 4-1(하위 개별)

Case 5: 혼합

구매대행 상품과 배송대행 상품이 하나의 패키지로 합쳐진 특수 케이스

  • Filter: 상위 패키지 내 'BUY_REQUEST/BUYFORME'와 'ASN'이 동시에 존재해야 함
  • Note: 순수 혼합이 아닌(기타 타입 포함) 건은 제외 처리

2. 케이스별 순이익(profit_krw) 계산 공식

[Case 1] (창고 서비스 수수료 11종 합계 * 시스템 환율) + 배송 마진(Mark-up)
[Case 2] (상품가 + 핸들링피 + PG수수료 + DK수수료 + 창고 수수료 + 배송 마진 + 부가세 환급) - (국내 배송비 + DK 할인 + 쿠폰)
[Case 3]
- 상위: (상위 전용 창고 수수료 합계 * 시스템 환율) + 배송 마진
- 하위(3-1): 하위 개별 창고 수수료 합계 * 시스템 환율
[Case 4]
- 상위: (하위 구매대행 상품 수익 합계) + (상위 통합 창고 수수료 * 시스템 환율) + 배송 마진
- 하위(4-1): 본인 상품 수익 + (본인 개별 창고 수수료 * 시스템 환율)
[Case 5]
- 상위: (하위 구매대행 상품 수익 합계) + (상위 통합 창고 수수료 * 시스템 환율) + 배송 마진
- 하위(5-1): 본인 상품 수익(구매대행인 경우만) + (본인 개별 창고 수수료 * 시스템 환율)
* 상품 수익 공식: (상품가 + 핸들링피 + 부가세환급 + 마켓수수료) - (할인액 + 쿠폰액)

3. 주요 데이터 컬럼(Column) 정의

컬럼명 설명
suite_number 고객 식별 번호 (Suite No.)
ship_date_kst 실제 국내 창고 출고 날짜 (KST)
trans_date_utc_package 패키지 결제/생성 날짜 (UTC)
package_id 최상위 패키지 고유 ID (Console/Repack인 경우 번호 동일)
inner_package_id 하위 개별 패키지 ID (Single은 NULL, 하위는 고유번호)
order_type 서비스 구분 (WE_SHIP: 배송대행, WE_BUY: 구매대행 등)
package_type 패키지 형태 (SINGLE, CONSOLE, REPACK, INNER)
actual_weight / dim_weight 실제 무게 및 부피 무게 (kg)
reference_type 등록 출처 (ASN: 사전통지, BUY_REQUEST/BUYFORME: 구매대행)
goods_revenue_krw 상품 관련 매출 합계 (환율 적용)
warehouse_revenue_krw 창고 서비스 수수료 11종 합계
shipping_revenue_krw 고객 결제 배송비 (shipfn.shipping_fee 기준)
marked_up_cost 배송 마진 (고객 결제 배송비 - 운송 원가)
profit_krw 최종 산출된 원화 기준 순이익
storage_fee / repack_fee 등 개별 부가 서비스 수수료 상세 (11종)

4. 데이터 집계 및 환율 정책

  • 중복 제거 정책: 동일 송장번호(tracking_number)에 대해 원가가 여러 번 잡히는 경우, ROW_NUMBER()를 사용하여 original_cost가 가장 높은 레코드를 기준으로 중복을 제거합니다.
  • 환율 적용: 패키지의 거래 발생일(trans_at_utc)을 기준으로 currency 테이블의 환율을 매칭합니다. 하루에 여러 환율이 존재할 경우 가장 높은 환율을 우선 적용합니다.
  • 시스템 환율 (1,450원): 데이터 상 환율 정보가 누락된 경우, 실질 수익 보수적 계산을 위해 기본 시스템 환율(1,450 KRW/USD)을 Default로 사용합니다.
  • 묶음 배송 부풀림 방지: 상위(Parent) 패키지와 하위(Inner) 패키지를 조인할 때 상위 배송비가 중복 합산되지 않도록 Case 4, 5에서는 사전 집계(Aggregated CTE)를 통해 데이터 무결성을 보장합니다.
  • 날짜 범위: 현재 대시보드의 모든 데이터는 2025-01-01부터 2026-02-20까지의 거래 데이터를 기준으로 합니다.
