1. 분석 대상 및 케이스 분류

Case 1: 배송대행 (Single)

배송대행(ASN) 서비스를 통해 들어온 단일 패키지 대상

  • Filter: package_type='SINGLE', reference_type='ASN'

Case 2: 구매대행 (Single)

구매대행(BUY_REQUEST) 서비스를 통해 결제 및 출고된 단일 패키지 대상

  • Filter: last_status != 'CANCELED', reference_type IN ('BUY_REQUEST', 'BUYFORME')

Case 3: 배송대행 (묶음)

배송대행 상품들로만 구성된 합배송(Console/Repack) 패키지

  • Filter: package_type IN ('CONSOLE', 'REPACK'), All Inner='ASN'
  • Structure: Case 3(상위) + Case 3-1(하위 개별)

Case 4: 구매대행 (묶음)

구매대행 상품들로만 구성된 합배송(Console/Repack) 패키지

  • Filter: package_type IN ('CONSOLE', 'REPACK'), All Inner='BUY_REQUEST/BUYFORME'
  • Structure: Case 4(상위 합산) + Case 4-1(하위 개별)

Case 5: 혼합

구매대행 상품과 배송대행 상품이 하나의 패키지로 합쳐진 특수 케이스

  • Filter: 상위 패키지 내 'BUY_REQUEST/BUYFORME'와 'ASN'이 동시에 존재해야 함

2. 케이스별 순이익(profit_krw) 계산 공식

[Case 1] 배송대행 SINGLE  package_type=SINGLE · reference_type=ASN
profit_krw = warehouse_revenue_krw + marked_up_cost_krw ※ goods_revenue = 0 (배송대행은 상품 구매 없음)
[Case 2] 구매대행 SINGLE  package_type=SINGLE · reference_type=BUY_REQUEST/BUYFORME
profit_krw = goods_revenue_krw + warehouse_revenue_krw + marked_up_cost_krw
  • ▸ goods_revenue_krw =
       상품가(×수량) + 핸들링피 + PG 수수료 + DK 수수료 + 부가세 환급
       − 국내 배송비 − 할인(discount) − 쿠폰 할인(coupon)
  • ▸ PG 수수료: 상품가의 7.78~8% (market_id별 차등, 일부 채널 0%)
  • ▸ DK 수수료: 상품가의 5~8% (market_id별 차등, 일부 채널 0%)
  • ▸ 부가세 환급: 상품가의 10% — B2B 거래 해당 market_id만 적용 (번장·포카마켓 등 C2C 제외)
  • ▸ warehouse_revenue_krw: 창고 수수료 11종 합계 × 시스템 환율(usd_krw)
  • ▸ marked_up_cost_krw: 고객 결제 배송비 − 배송 원가
[Case 3] 배송대행 CONSOLE/REPACK  모든 하위 패키지 reference_type=ASN
  • ▸ Case 3 (상위): warehouse_revenue_krw + marked_up_cost_krw
  • ▸ Case 3-1 (하위 개별): 하위 패키지별 창고 수수료 11종 × 시스템 환율(usd_krw)
    ※ 배송 마진(marked_up_cost)은 상위에만 포함, 하위 미포함
[Case 4] 구매대행 CONSOLE/REPACK  모든 하위 패키지 reference_type=BUY_REQUEST/BUYFORME
  • ▸ Case 4 (상위): Σ(하위 상품 수익) + warehouse_revenue_krw + marked_up_cost_krw
    ※ 하위 goods_revenue 전체 합산 후 상위의 창고 수수료 · 배송 마진 추가
  • ▸ Case 4-1 (하위 개별): 개별 goods_revenue + 개별 창고 수수료 11종 × 시스템 환율(usd_krw)
[Case 5] 혼합 (배송+구매대행 합패키지)  상위 내 ASN + BUY_REQUEST/BUYFORME 혼재
  • ▸ Case 5 (상위): Σ(구매대행 하위 상품 수익) + warehouse_revenue_krw + marked_up_cost_krw
    ※ 배송대행 하위는 goods_revenue=0이므로 구매대행 하위만 합산
  • ▸ Case 5-1 (하위 개별):
    · 구매대행 하위: 개별 goods_revenue + 창고 수수료 11종 × 시스템 환율(usd_krw)
    · 배송대행 하위: 창고 수수료 11종 × 시스템 환율(usd_krw)만 (goods_revenue=0)
▸ goods_revenue 공식 (Case 2 · 4-1 · 5-1 공통)
  = 상품가(×수량) + 핸들링피 + PG 수수료 + DK 수수료 + 부가세 환급 − 국내 배송비 − 할인 − 쿠폰
▸ 환율 적용: trans_at_utc 기준 시스템 환율(currency 테이블의 usd_krw 컬럼). 누락 시 1,450 KRW/USD
▸ PG · DK 수수료 요율: market_id별 차등 → 5. market_id 분류 참조

4. 데이터 집계 및 환율 정책

  • 중복 제거 정책: 동일 송장번호(tracking_number)에 대해 원가가 여러 번 잡히는 경우, ROW_NUMBER()를 사용하여 original_cost가 가장 높은 레코드를 기준으로 중복을 제거합니다.
  • 환율 적용: 패키지의 거래 발생일(trans_at_utc)을 기준으로 currency 테이블의 환율을 매칭합니다. 하루에 여러 환율이 존재할 경우 가장 높은 환율을 우선 적용합니다.
  • 시스템 환율 (1,450원): 데이터 상 환율 정보가 누락된 경우, 실질 수익 보수적 계산을 위해 기본 시스템 환율(1,450 KRW/USD)을 Default로 사용합니다.
  • 묶음 배송 부풀림 방지: 상위(Parent) 패키지와 하위(Inner) 패키지를 조인할 때 상위 배송비가 중복 합산되지 않도록 Case 4, 5에서는 사전 집계(Aggregated CTE)를 통해 데이터 무결성을 보장합니다.
  • 날짜 범위: 현재 대시보드의 모든 데이터는 배송비 결제일 기준 2025-01-01부터 2026-02-20까지의 거래 데이터를 기준으로 합니다.
